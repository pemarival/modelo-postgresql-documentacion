# Título
Adopción de Docker para empaquetado y despliegue del sistema.

# Contexto
El sistema incluye dos componentes:
- base de datos PostgreSQL
- versionamiento de la base de datos por medio de liquibase
  
Configurar manualmente estos componentes en distintos entornos puede generar inconsistencias.

# Problema
Los entornos de desarrollo y despliegue pueden diferir en:
- versiones de software
- configuraciones del sistema
- dependencias

Esto puede generar errores como:
“funciona en mi máquina pero no en producción”.

# Decisión 
Se decide utilizar Docker para contenedorización del sistema.

Cada componente será desplegado mediante contenedores.

Ejemplo de arquitectura:
docker-compose
 ├── liquibase
 ├── postgres-db
 
# Justificación técnica
Docker permite:
- portabilidad entre entornos
- despliegues reproducibles
- aislamiento de dependencias
- integración sencilla con pipelines CI/CD

Además, facilita el escalamiento futuro mediante orquestadores como Kubernetes.

# Consecuencias o impacto esperado
### Positivo
- entornos consistentes entre desarrollo y producción
- despliegues más rápidos
- facilidad para escalar servicios
  
### Negativo
- curva de aprendizaje en contenedorización
- necesidad de gestionar imágenes y registros de contenedores
