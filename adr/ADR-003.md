# Título
Adopción de Liquibase como herramienta para versionar el esquema de base de datos.

# Contexto
El sistema posee un modelo de datos amplio con varias tablas relacionadas que soportan múltiples dominios funcionales.

Durante el desarrollo, el esquema de base de datos puede evolucionar constantemente debido a:
- cambios en requisitos
- correcciones en el modelo
- optimizaciones de rendimiento
  
# Problema
Gestionar cambios en el esquema de base de datos manualmente puede generar:
- inconsistencias entre entornos
- pérdida de control sobre versiones del esquema
- dificultad para desplegar cambios en ambientes de QA o producción
  
# Decisión 
Se decidió utilizar Liquibase como herramienta de control de versiones para el desarrollo del sistema.

Liquibase permitirá gestionar:
- creación de tablas
- alteraciones de esquema
- índices
- restricciones

mediante archivos versionados en el repositorio del proyecto.

# Justificación técnica
Liquibase ofrece:

- versionamiento estructurado del esquema
- ejecución controlada de migraciones
- compatibilidad con múltiples motores de base de datos
- registro histórico de cambios aplicados

Esto facilita la sincronización entre entornos:
- desarrollo
- QA
- producción
  
# Consecuencias o impacto esperado
### Positivo
- control total sobre evolución del esquema
- facilidad para replicar entornos
- reducción de errores en despliegues
  
### Negativo
- curva de aprendizaje inicial para el equipo
- necesidad de disciplina en el manejo de cambios de esquema
